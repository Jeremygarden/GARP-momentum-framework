# 执行SOP（标准作业流程）

> **核心架构原则**（不可随意修改）
> 1. **风险因子（R）全程固定 10%**，不参与动态调整，不参与线性插值
> 2. 风险因子承担双重角色：①常规加减分 ②一票否决权（触发即归零，与权重无关）
> 3. **PEG阈值和股息率加分参数随市场状态切换即时生效**（无缓冲），权重切换有5日缓冲
> 4. ERP < 0 时无论其他信号，立即触发全面防御，暂停新建仓
> 5. **【2026-04-02 新增，强制执行】每次刷新必须执行广度扫描，不得仅在原15只上重算**（见步骤4.5）

## 步骤0：市场状态判断（每周一，执行一次）

**数据获取（L3 tavily_extract 为主; 如果失败报错以mx-data skill为辅）：**

| 指标 | 数据源 |
|------|--------|
| 全A PE(TTM) 当前值 | 东方财富全A估值页 |
| 全A PE 10年历史百分位 | 手动记录表 / Wind |
| 北向10日累计净流入 | 东方财富北向资金页 |
| 上证指数20日历史波动率 | thsdk / 同花顺 |
| 10年国债收益率 | 中债官网 |
| ERP = 1/PE(全A) − 10年国债收益率 | 计算得出 |

**四维投票判断 → 确认市场状态（见 `market-regime.md`）：**

```
≥3个变量指向同一状态 → 确认切换
2:2分裂 → 以 PE百分位 + ERP 为主裁
```

**关键规则：**
```
□ ERP < 0 → 立即触发全面防御，暂停新建仓，持仓只减不增
□ 状态切换：权重切换有5日缓冲（线性过渡），PEG阈值/股息率参数即时切换
□ 状态切换事件写入 Bitable 日志（字段：日期/前状态/后状态/触发指标值）
□ 全A PE历史数据缺失时，使用沪深300 PE TTM代替
```

## 步骤1：数据获取链路（七级优先，按序执行，成功即止）

> **复权标准（强制）**：所有历史价格数据必须使用**前复权（QFQ）**
> - baostock：`adjustflag="2"` ← 必须显式指定，默认"3"=不复权
> - thsdk：`adjust="前复权"` ← 必须显式指定，默认不复权
> - yfinance：`auto_adjust=True`（默认，无需额外操作）
> - MX 系列：返回实时/基本面数据，无历史复权问题

```
L1 MX-StockPick（市值/PE/PB/涨跌幅等基础行情）
   → 适用：V估值初始筛选、G成长初筛
   → 工具：scripts/mx_select_stock.py "{条件}"

L2 MX-Data（深度财务：ROE/净利润/EPS/现金流/估值历史）
   → 适用：G/Q/V 核心财务因子
   → 工具：scripts/mx_data.py "{股票名} ROE净利润2025增速"

L3 MX-FinSearch（最新券商研报 · 11家重点公司批量抓取）
   → 适用：一致预期 EPS / 盈利修正动量 / 分析师评级
   → 工具：scripts/mx_search.py "{股票名}最新研报"

L4 baostock / thsdk（A股历史价格，前复权）
   → 适用：M动量价格计算（6M超额收益）
   → baostock: adjustflag="2"（前复权）
   → thsdk: adjust="前复权"

L5 yfinance（港股/美股历史价格，默认前复权）
   → 适用：港股标的 M动量（如 9992.HK / 700.HK / 6181.HK）

L6 Tavily L3/L4（网搜研报补充 · 宏观趋势数据）
   → 适用：L1-L5 覆盖不足时的补充验证
   → 宏观数据（如中国集成电路产量/出口月度数据）：MX-MacroData 或 Tavily

L7 中性填补（最后兜底）
   → 必须标注 ⚠️，记入数据缺口表，置信度标记为 ⭐⭐⭐ 以下
```

**数据刷新频率（分层策略）：**

| 因子 | 刷新频率 | 数据源 | 说明 |
|------|---------|--------|------|
| **M 动量** | **每周一**（每次执行）| L1 thsdk 实时量价 | 价格动量每周必须刷新 |
| **G/Q/V 基本面** | **2.1 / 5.1 / 8.1 / 11.1** | L1-L3 完整拉取 | 与季报披露对齐 |
| **一致预期数据** | **2.1 / 4.30 / 7.15 / 10.15** | L3-L4 研报 | 取最近 90-150天有效均值，超期数据降级⚠️ |

> 非刷新周期内，G/Q/V 使用上次缓存值，仅刷新 M，总分据此重新计算排名。

## 步骤2：初筛（每季度或重大行情后，广度过滤）

```
基础过滤条件（全部满足）：
  □ 行业TAM > 750亿，未来3年CAGR > 15%
  □ 资产负债率 < 70%（非金融）
  □ 商誉/净资产 < 35%
  □ 控股股东质押比 < 60%
  □ 近3年经营现金流非连续为负

预期：500-2000家 → 50-150家候选
```

## 步骤3：赛道质量复选（深度，50-150家 → 20-50家）

```
按行业分组，对每家公司评估：
  1. CR3（行业集中度）
  2. 供给扩张比（产能CAGR vs 需求CAGR）
  3. 行业产品价格趋势（过去12个月）

赛道质量评分 < 43分 → 降级或剔除
```

## 步骤4：五因子评分（精度，20-50家逐一打分）

**⚠️ 权重加载规则（必读）：**
```
W1-W4（G/Q/V/M）从 market-regime.md 按当前状态加载，合计 90%
W5（风险R）始终固定 10%，不随状态变化
```

**评分计算：**
```
[成长 G]
  G_adjusted = min(历史3年CAGR×0.70, 券商预期×0.80, 行业CAGR)
  成长总分 = 赛道质量评分 × 0.4 + G_adjusted得分 × 0.6

[质量 Q]
  按行业分组选择对应质量基准（layer2-quality.md）
  股息率附加分：按当前市场状态加载加减分参数（market-regime.md §6.2）

[估值 V]
  PEG_adjusted = PE(TTM) / G_adjusted(%)
  PEG评分：按当前市场状态加载 PEG阈值（market-regime.md §6.1）
  超过当前状态可接受上限 → V分 ≤ 40

[动量 M]
  价格动量（6M超额收益）× 55% + 盈利修正动量（3M预期变化）× 45%

[风险 R，固定10%]
  一票否决检查（触发即总分归零，与权重无关）：
    □ 控股股东质押 > 60%
    □ 近3年经营现金流连续为负
    □ 商誉/净资产 > 35%
    □ 财务造假迹象 / 退市风险警示
  常规加减分：±5-10分

  R 分含义：分越高 = 风险健康度越高（越安全）
    90-100：财务干净，无集中风险，治理优秀
    80-89 ：有1-2个需关注点，不触发否决，但需持续监控
    60-79 ：中等风险，减仓预警区，优先核查核心论点
    < 60  ：高风险，考虑减仓；< 40 触发一票否决归零

  ⚠️ R < 90 强制输出"风险披露"规则：
    每次榜单输出时，对 R < 90 的标的必须额外列出以下四项：
    ① 风险等级标注（如 R=80 → ⚠️ 中低风险）
    ② 风险背后含义（具体是什么风险，1-2句说清楚）
    ③ 相关实证数据（来自最近一期财报/公告的具体数字）
    ④ 复查建议（下次复查重点看什么指标）

    示例（老铺黄金 R=80）：
      ⚠️ 风险等级：中低风险（R=80）
      风险含义：快速扩张期现金流承压，负债结构恶化
      实证数据：2025年末存货+292%，短期借款+348%，经营现金流约-53亿
      复查建议：下期关注经营现金流是否转正、短借/营收比是否收窄

总分 = G×W1 + Q×W2 + V×W3 + M×W4 + R×0.10
梯队：> 85分=钻石 / 75-84=黄金 / 60-74=白银 / <60=剔除
```

## 步骤4.5：广度扫描层（每次刷新强制执行，不可跳过）

> **【2026-04-02 新增】** 触发背景：2026-04-02临时刷新因跳过此步骤，导致三星医疗（77.2分）漏选。
> 
> **原则：就算被挤出 Top15，也要进观察池/候补池，永不直接忽略重要标的。**

### 4.5.1 广度扫描覆盖要求

每次刷新不得仅在上期 Top15 上重算，必须同时覆盖以下范围：

```
① 当前 Top15 标的（重算所有因子）
② 观察池全部标的（重算 M 分 + 总分）
③ 候补池全部标的（检查是否有升格至 75+ 的标的）
④ 赛道代表性扫描：
   - 每个主要赛道取3-5只代表性标的重新对比评分
   - 赛道范围：AI算力/光通信、电力设备/电网、储能/新能源、
     工控/机器人、港股科技消费、医疗器械、消费IP等
   - 目标：捕捉赛道内未在榜但评分已升至 75+ 的新标的
⑤ 市场热点赛道新进入者：当周成交量/研报覆盖突增的赛道，
   增加1-3只未曾评估的潜在标的
```

### 4.5.2 扫描后分流规则

```
评分结果 → 处理方式
≥ 75分   → 直接参与 Top15 竞争，按总分排序决定是否挤入
60-74分  → 无论当前榜单情况，写入观察池或候补池
40-59分  → 候补池备注，季度全量扫描时重新评估
< 40分 / 一票否决 → 记录剔除理由，本季不再扫描
```

### 4.5.3 漏选检查（每次输出榜单前必做）

```
□ 必扫名单中每只标的都已评分？
□ 候补池中有无已达75+但未进Top15的标的（应升入观察池）？
□ 观察池中有无M分回升、总分突破75的标的（应升入Top15候选）？
□ 本次新进的黑马来源是否可追溯（不是无中生有）？
```

## 步骤5：组合构建与买入纪律

```
仓位规则：
  □ 单一个股上限：8%（牛市泡沫期降至5%）
  □ 单一赛道上限：25%
  □ 总股票池：10-15只

买入时机（按当前状态的PEG阈值判断，market-regime.md §6.1）：
  首选：PEG_adjusted ≤ 当前状态"最优门槛"
  次选：PEG ≤ 当前状态"可接受上限" + 技术面右侧企稳
  动量确认：盈利修正上调 + 价格动量 > 行业中位数

禁止：
  □ 追高（最近1个月涨幅 > 20%）
  □ 市场状态切换5日缓冲期内重仓新建
  □ ERP < 0 期间新建任何仓位
```

## 步骤5.5：技术确认层（买卖时机过滤器）

> **定位**：不进入五因子评分，不影响排名，只用于判断"现在是否合适入场/离场"。
> GARP 回答"买什么"，技术确认层回答"什么时候买"。两者不混淆，互相确认。

### 买入确认信号（满足任一即可入场）
```
□ 价格站回 20日均线 且 当日成交量 > 5日均量（量价共振确认支撑）
□ MACD 日线金叉（DIF 上穿 DEA，短期动能转正）
□ RSI 从超卖区（< 30）回升至 35 以上（超跌反弹确认）
```

### 拒绝入场信号（满足任一则等待）
```
□ 仍处于连续下跌通道（5日内收盘价持续创近期新低）
□ 成交量持续萎缩（抛盘未止，不宜抄底）
□ 价格创近期新低且无基本面变化解释（底部未确认）
```

### 减仓/离场确认信号（满足任一）
```
□ GARP 总分跌破 75（黄金→白银，触发步骤6减仓规则）
□ PEG 超出当前状态可接受上限（market-regime.md §6.1）
□ 价格跌破 20日均线 且 成交量放大（趋势转弱）
□ 一票否决条件触发（立即清仓，不等技术确认）
```

### 暴跌场景标准分析流程
```
Step 1：判断基本面是否变化
  → 核查 G/Q 因子：ROE是否下降？增速是否骤降？论点是否被推翻？
  → 若未变化 → 进入 Step 2
  → 若已变化（业绩暴雷/ROE崩）→ 执行减仓，不是买入时机

Step 2：重新计算 PEG
  → 股价暴跌 → PE 降低 → PEG 可能重回合理区间
  → 若 PEG 回到当前状态"最优门槛"以内 → 标记为观察池候选

Step 3：等待技术右侧确认
  → 不抄刀：等 RSI 企稳或 MACD 金叉再入场
  → 5个交易日内出现买入确认信号 → 可建仓，将标的从观察池升入核心池候选

Step 4：纳入观察池（见步骤8.5）
  → 基本面未崩但总分暂时跌出 Top15 → 进入观察池双周跟踪
```

## 步骤6：持仓管理与退出规则

```
持续持有条件（全部满足）：
  □ 核心赛道景气 + 质量护城河论点未变
  □ 盈利修正未连续下调2个季度
  □ 价格动量未持续跑输行业中位数 > 15% 超3个月

主动减仓触发（满足任意一条）：
  □ 总分 < 60分（降至白银以下）
  □ 赛道质量触发重大降级（供给扩张比 > 1.8 / CR3快速下降）
  □ 一票否决条件触发 → 立即清仓
  □ 市场状态切换至"深度熊市" → 组合总仓位降至50%以下
  □ PE(TTM) 超过行业绝对上限30%
```

## 步骤7：Top15 榜单刷新与输出规则

> **定时执行：每周一 UTC 02:00（北京时间 10:00），范围：A股沪深300 + 港股恒生核心成分股**

### 7.1 排名变化标记规则

| 情况 | 标记方式 |
|------|---------|
| 排名上升 N 位 | 排名后加 `🔼+N`（如 `#3 🔼+2`）|
| 排名下降 N 位 | 排名后加 `🔽-N`（如 `#7 🔽-1`）|
| 排名无变化 | 不标记 |
| 新进入榜单（黑马）| 加 `🌟` 前缀（见 7.2）|

**标记适用范围：无论什么原因导致的名次变化（含黑马挤压），均做升降标记。**
唯一例外：被黑马挤压但**自身评分无任何变化**的标的，标注说明"因新晋黑马位移"但不加 🔼🔽 符号。

### 7.2 黑马定义与标注规则

**黑马定义：** 上次 Top15 榜单中不存在、本次新进入 Top15 的标的，无论是：
- 从未被跟踪的全新标的
- 曾经在榜但已跌出榜单、本次重新进入的标的

均算当月黑马，用 `🌟` 标记当前排名位置，并附：

```
🌟 [股票名]（代码）  第X名
入榜实证：[最近1个季报亮眼数据 / 重大事件，如"2026Q1净利润+X%超预期"]
触发因子：[GARP框架中的核心触发点，如"PEG降至0.6进入黄金区 / ROE突破20%"]
```

### 观察池纳入条件（满足任一）
```
□ 总分跌出 Top15（< 75）但 G分 + Q分合计 > 150（基本面健康，仅M或V拉分）
□ 总分 > 75 但 R < 75（高风险标的，需持续监控论点是否变化）
□ 刚经历暴跌（单日 > 10%）且无基本面恶化信号（技术面待确认右侧）
□ 季报发布后基本面待验证期（入榜论点核心数据尚未确认）
```

### 观察池退出条件
```
升级至核心池：M分恢复 + 总分回升 > 75 → 下次周刷新纳入 Top15

```
