> 版本：v1.0 | 制定日期：2026-04-06
> 本文档定义 **GARP 五因子评分体系、数据获取链路、权重计算、持仓管理和输出规范**。
> 不包含池子扫描频率和入池规则，这些内容见 → `docs/garp-pool-scan-rules.md`
> 不包含观察池入池判定条件，这些内容见 → `docs/GARP-观察池入选规则-v1.1.md`

## 一、架构原则（不可随意修改）

1. **风险因子（R）全程固定 10%**，不参与动态调整，不参与线性插值
2. 风险因子承担双重角色：①常规加减分 ②一票否决权（触发即总分归零，与权重无关）
3. **PEG 阈值和股息率加分参数随市场状态切换即时生效**（无缓冲）；权重切换有 5 日缓冲（线性过渡）
4. **ERP < 0** 时无论其他信号，立即触发全面防御，暂停新建仓，持仓只减不增

## 二、数据获取链路（七级优先，按序执行，成功即止）

### 复权标准（全局强制）

> **所有历史价格数据必须使用前复权（QFQ）**。动量因子、技术确认层、超额收益计算均基于前复权价格。

| 数据源 | 复权行为 | 正确参数 |
|--------|:--------:|---------|
| baostock | 需手动指定 | `adjustflag="2"`（1=后复权, **2=前复权** ✅, 3=不复权）|
| thsdk | 需手动指定 | `adjust="前复权"`（空字符串=不复权）|
| yfinance | 默认前复权 | `auto_adjust=True`（默认，无需额外操作）✅ |
| MX-Data / MX-StockPick | 实时/基本面数据 | 不涉及历史复权 ✅ |

### 七级优先链路

```
L1 MX-StockPick（市值/PE/PB/涨跌幅等基础行情）
   → 适用：V估值初始筛选、G成长初筛
   → 工具：scripts/mx_select_stock.py "{条件}"

L2 MX-Data（深度财务：ROE/净利润/EPS/现金流/估值历史）
   → 适用：G/Q/V 核心财务因子
   → 工具：scripts/mx_data.py "{股票名} ROE净利润2025增速"

L3 MX-FinSearch（最新券商研报，11家重点公司批量抓取）
   → 适用：一致预期 EPS / 盈利修正动量 / 分析师评级
   → 工具：scripts/mx_search.py "{股票名}最新研报"

L4 baostock / thsdk（A股历史价格，前复权）
   → 适用：M动量价格计算（6M超额收益）
   → baostock: adjustflag="2" | thsdk: adjust="前复权"

L5 yfinance（港股/美股历史价格，默认前复权）
   → 适用：港股标的 M动量（如 9992.HK / 700.HK / 6181.HK）

L6 Tavily（网搜研报补充 · 宏观趋势数据）
   → 适用：L1-L5 覆盖不足时的补充验证
   → 宏观数据：MX-MacroData 或 Tavily

L7 中性填补（最后兜底）
   → 必须标注 ⚠️，记入数据缺口表，置信度标记为 ⭐⭐⭐ 以下
```

### 数据刷新频率（分层策略）

| 因子 | 刷新频率 | 数据源 | 说明 |
|------|---------|--------|------|
| M 动量 | 每周一（每次执行）| L4/L5 历史价格 | 价格动量每周必须刷新 |
| G/Q/V 基本面 | 2.1 / 5.1 / 8.1 / 11.1 | L1–L3 完整拉取 | 与季报披露对齐 |
| 一致预期数据 | 2.1 / 4.30 / 7.15 / 10.15 | L3–L4 研报 | 取最近 90–150天有效均值，超期数据降级⚠️ |

> 非刷新周期内，G/Q/V 使用上次缓存值，仅刷新 M，总分据此重新计算排名。

## 三、五因子评分体系

### 步骤 A：市场状态判断（每周一，执行一次）

获取以下指标，通过四维投票判断当前市场状态：

| 指标 | 数据源 |
|------|--------|
| 全A PE(TTM) 当前值 | 东方财富全A估值页（L6 Tavily 为主，失败则 MX-Data 为辅）|
| 全A PE 10年历史百分位 | 手动记录表 / Wind |
| 北向10日累计净流入 | 东方财富北向资金页 |
| 上证指数20日历史波动率 | thsdk / 同花顺 |
| 10年国债收益率 | 中债官网 |
| ERP = 1/PE(全A) − 10年国债收益率 | 计算得出 |

投票规则：
```
≥3个变量指向同一状态 → 确认切换
2:2 分裂 → 以 PE百分位 + ERP 为主裁
```

关键规则：
```
□ ERP < 0 → 立即全面防御，暂停新建仓，持仓只减不增
□ 状态切换：权重切换有5日缓冲（线性过渡），PEG阈值/股息率参数即时切换
□ 状态切换事件写入 Bitable 日志（日期/前状态/后状态/触发指标值）
□ 全A PE历史数据缺失时，使用沪深300 PE TTM代替
```

📄 状态定义与对应参数：`references/garp-quality-momentum/market-regime.md`

### 步骤 B：初筛（每季度或重大行情后）

```
基础过滤条件（全部满足）：
  □ 行业 TAM > 750亿，未来3年 CAGR > 15%
  □ 资产负债率 < 70%（非金融）
  □ 商誉/净资产 < 35%
  □ 控股股东质押比 < 60%
  □ 近3年经营现金流非连续为负

预期：500–2000家 → 50–150家候选
```

### 步骤 C：赛道质量复选（50–150家 → 20–50家）

```
按行业分组，对每家公司评估：
  1. CR3（行业集中度）
  2. 供给扩张比（产能CAGR vs 需求CAGR）
  3. 行业产品价格趋势（过去12个月）

赛道质量评分 < 43分 → 降级或剔除
```

### 步骤 D：五因子评分（精度，20–50家逐一打分）

**权重加载规则（必读）：**
```
W1–W4（G/Q/V/M）从 market-regime.md 按当前状态加载，合计 90%
W5（风险R）始终固定 10%，不随状态变化
```

#### [G] 成长性

```
G_adjusted = min(历史3年CAGR × 0.70, 券商预期 × 0.80, 行业CAGR)
成长总分 = 赛道质量评分 × 0.4 + G_adjusted得分 × 0.6
```

#### [Q] 质量因子

```
按行业分组选择对应质量基准
股息率附加分：按当前市场状态加载参数（market-regime.md §6.2）
```
📄 行业质量基准：`references/garp-quality-momentum/layer2-quality.md`

#### [V] 估值合理性

```
PEG_adjusted = PE(TTM) / G_adjusted(%)
PEG评分：按当前市场状态加载 PEG阈值（market-regime.md §6.1）
超过当前状态可接受上限 → V分 ≤ 40
```

#### [M] 价格动量

```
M = 价格动量（6M超额收益）× 55% + 盈利修正动量（3M预期变化）× 45%
```

#### [R] 风险因子（固定权重 10%）

**一票否决（触发即总分归零，与权重无关）：**
```
□ 控股股东质押 > 60%
□ 近3年经营现金流连续为负
□ 商誉/净资产 > 35%
□ 财务造假迹象 / 退市风险警示
```

**R 分含义（分越高 = 风险健康度越高）：**

| R 分 | 含义 |
|:----:|------|
| 90–100 | 财务干净，无集中风险，治理优秀 |
| 80–89 | 有1-2个需关注点，不触发否决，需持续监控 |
| 60–79 | 中等风险，减仓预警区，优先核查核心论点 |
| < 60 | 高风险，考虑减仓；< 40 触发一票否决归零 |

**⚠️ R < 90 强制输出"风险披露"规则：**

每次榜单输出时，对 R < 90 的标的必须额外列出：
1. 风险等级标注（如 R=80 → ⚠️ 中低风险）
2. 风险背后含义（具体是什么风险，1–2句）
3. 相关实证数据（来自最近一期财报/公告的具体数字）
4. 复查建议（下次复查重点看什么指标）

示例（老铺黄金 R=80）：
```
⚠️ 风险等级：中低风险（R=80）
风险含义：快速扩张期现金流承压，负债结构恶化
实证数据：2025年末存货+292%，短期借款+348%，经营现金流约-53亿
复查建议：下期关注经营现金流是否转正、短借/营收比是否收窄
```

#### 总分计算

```
总分 = G×W1 + Q×W2 + V×W3 + M×W4 + R×0.10

梯队划分：
  > 85分  → 💎 钻石级（核心底仓）
  75–84分 → 🥇 黄金级
  60–74分 → 🥈 白银级
  < 60分  → 剔除
```

## 四、买入纪律与持仓管理

### 仓位规则

```
□ 单一个股上限：8%（牛市泡沫期降至5%）
□ 单一赛道上限：25%
□ 总股票池：10–15只
```

### 买入时机

```
首选：PEG_adjusted ≤ 当前状态"最优门槛"（market-regime.md §6.1）
次选：PEG ≤ 当前状态"可接受上限" + 技术面右侧企稳
动量确认：盈利修正上调 + 价格动量 > 行业中位数

禁止：
  □ 追高（最近1个月涨幅 > 20%）
  □ 市场状态切换5日缓冲期内重仓新建
  □ ERP < 0 期间新建任何仓位
```

### 技术确认层（买卖时机过滤器）

> GARP 回答"买什么"，技术确认层回答"什么时候买"。两者互相确认，不混淆评分。

**买入确认信号（满足任一即可入场）：**
```
□ 价格站回20日均线 且 当日成交量 > 5日均量（量价共振确认支撑）
□ MACD 日线金叉（DIF 上穿 DEA，短期动能转正）
□ RSI 从超卖区（< 30）回升至 35 以上（超跌反弹确认）
```

**拒绝入场信号（满足任一则等待）：**
```
□ 仍处于连续下跌通道（5日内收盘价持续创近期新低）
□ 成交量持续萎缩（抛盘未止，不宜抄底）
□ 价格创近期新低且无基本面变化解释（底部未确认）
```

**减仓/离场确认信号（满足任一）：**
```
□ GARP 总分跌破 75（黄金→白银，触发减仓规则）
□ PEG 超出当前状态可接受上限（market-regime.md §6.1）
□ 价格跌破20日均线 且 成交量放大（趋势转弱）
□ 一票否决条件触发（立即清仓，不等技术确认）
```

**暴跌场景标准分析流程：**
```
Step 1：判断基本面是否变化
  → 若已变化（业绩暴雷/ROE崩）→ 执行减仓，不是买入时机
  → 若未变化 → 进入 Step 2

Step 2：重新计算 PEG
  → 股价暴跌 → PE降低 → PEG可能重回合理区间
  → 若 PEG 回到当前状态"最优门槛"以内 → 标记为观察池候选

Step 3：等待技术右侧确认
  → 不抄刀：等 RSI 企稳或 MACD 金叉再入场

Step 4：纳入观察池跟踪
  → 📄 入池规则：`docs/GARP-观察池入选规则-v1.1.md` §二·路径A
```

### 持仓维持条件（全部满足）

```
□ 核心赛道景气 + 质量护城河论点未变
□ 盈利修正未连续下调2个季度
□ 价格动量未持续跑输行业中位数 > 15% 超3个月
```

### 主动减仓触发（满足任一）

```
□ 总分 < 60分（降至白银以下）
□ 赛道质量触发重大降级（供给扩张比 > 1.8 / CR3快速下降）
□ 一票否决条件触发 → 立即清仓
□ 市场状态切换至"深度熊市" → 组合总仓位降至50%以下
□ PE(TTM) 超过行业绝对上限30%
```
