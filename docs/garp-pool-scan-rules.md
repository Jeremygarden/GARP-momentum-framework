## 二、核心池 Top15 — 周度增量刷新

**触发：** Cron `garp-top15-weekly-refresh`，每周一 UTC 02:00
**执行者：** Investor Agent（isolated）

### 步骤

**Step 1：数据更新**
- 拉取 Top15 各标的最新数据
- 📄 数据获取链路：`docs/garp-core-framework.md` — **数据获取链路**
- 按季度刷新策略判断本次是否完整刷新 G/Q/V，还是仅刷新 M 动量

**Step 2：五因子重新评分**
- 📄 评分计算方法：`docs/garp-core-framework.md` — **五因子评分体系**
- 加载当前市场状态对应权重
- 📄 市场状态与权重：`references/garp-quality-momentum/market-regime.md`

**Step 3：广度扫描（强制，不可跳过）**

> **【2026-04-02 强制新增】** 每次刷新必须执行广度扫描，不得仅在原15只上重算。
> 背景：2026-04-02 临时刷新跳过此步，导致三星医疗（77.2分）漏选。
> 原则：就算被挤出 Top15，也要进观察池/候补池，永不直接忽略重要标的。

扫描覆盖范围：
```
① 当前 Top15 标的（重算所有因子）
② 观察池全部标的（重算 M 分 + 总分）
③ 候补池全部标的（检查是否有升格至 75+ 的标的）
④ 赛道代表性扫描：
   每个主要赛道取 3-5 只代表性标的对比评分
   赛道范围：AI算力/光通信、电力设备/电网、储能/新能源、
             工控/机器人、港股科技消费、医疗器械、消费IP等
⑤ 市场热点赛道新进入者：当周研报覆盖/成交量突增赛道，
   增加 1-3 只未曾评估的潜在标的
```

扫描后分流规则：
```
评分     → 处理方式
≥ 75分  → 参与 Top15 竞争，按总分排序决定是否挤入
60-74分 → 写入观察池或候补池（二选一，见下方 **观察池** 和 **候补池** 章节）
40-59分 → 候补池备注，季度全量扫描时重新评估
< 40分 / 一票否决 → 记录剔除理由，本季不再扫描
```

漏选检查（每次输出榜单前必做）：
```
□ 必扫名单中每只标的都已评分？
□ 候补池中有无已达 75+ 但未进 Top15 的标的（应升入观察池）？
□ 观察池中有无 M 分回升、总分突破 75 的标的（应升入 Top15 候选）？
□ 本次新进的黑马来源是否可追溯（不是无中生有）？
```

**Step 4：Top15 降级检查**
- 对每只 Top15 标的逐一检查降级条件
- 📄 降级条件（A1急性冲击 / A2慢性预警）：`docs/GARP-观察池入选规则-v1.1.md` — **路径A**:Top15降级
- 若触发 A1：当周立即降级，移入观察池
- 若触发 A2：在 Bitable 打 `⚠️预警` 标签，下次双周复查时确认

**Step 5：排名重算与输出**
- 按总分重新排序，更新"本期排名"、"排名变化"字段
- 📄 输出格式：`docs/garp-core-framework.md` — **榜单输出格式**

**Step 6：完成通知**
- 向 Feishu 群发送刷新摘要（新增/移除/排名变化/当月黑马）

## 三、观察池 Watch — 双周复查

**触发：** 每逢偶数周（含核心池刷新时附带执行）
**执行者：** Investor Agent

### 入池路径
📄 完整入池规则：`docs/GARP-观察池入选规则-v1.1.md` — **入池路径A/B**

快速参考：
- **路径 A1**：Top15 急性冲击降级（动量骤降/暴跌/风险突变）
- **路径 A2**：Top15 慢性恶化预警确认后降级（Q/G/V/竞争/股东行为）
- **路径 B**：候补池触发强信号（单条）或弱信号（任意两条叠加）晋升

### 双周复查清单

对每只观察池标的：
```
□ M 分变化（技术面是否修复）
□ 最新季报/月度数据（G 分是否稳定）
□ ROE 趋势（Q 分是否恶化）
□ A2 预警标签中的指标是否改善或继续恶化
□ 升级条件是否已满足
□ 是否触发移除条件
```

### 升级至 Top15 条件（同时满足）

| 条件 | 路径A（降级修复）| 路径B（候补晋升）|
|------|:---:|:---:|
| M 分 | ≥ 60 | ≥ 75 |
| 技术面 | RSI 企稳 + MACD 金叉 | RSI 企稳 + MACD 金叉 |
| 基本面验证 | 季报增速符合入池预期 | 季报增速符合入池预期 |
| PEG | < 1.5 | < 1.5 |
| 替换逻辑 | 综合评分 > Top15 末位 | 综合评分 > Top15 末位 |

### 移除条件（满足任一）
```
□ ROE 跌破 12%（实质性基本面恶化）
□ 净利润连续两季负增长
□ 行业政策重大变化（监管打压、集采覆盖核心产品）
□ 在观察池超过 3 个月仍未满足升级条件
□ 已晋升核心池 Top15
```

### 输出格式（双周发至 Feishu）
```
👁 GARP 观察池 | {日期}
━━━━━━━━━━━━━━━
👁 [股票名] [代码]  [G=x Q=x V=x M=x R=x] 总分≈xx
   纳入路径：路径A1 / A2 / B
   状态：[描述]
   待验证：[具体事项]
   技术：[等待右侧 / RSI企稳 / MACD金叉]
   升级条件：[具体描述]
   A2预警：[若有：⚠️ ROE连续下行 / ⚠️ PEG泡沫化 等]
━━━━━━━━━━━━━━━
```

## 四、候补池 Pipeline — 月度全量扫描

**触发：** 每月 1 日，手动触发（Cron 待注册）
**执行者：** Investor Agent

### 硬门槛初筛（全部满足方可进入候补池）

```
基本面门槛：
  □ 净利润增速 ≥ 20%（近一年）
  □ ROE ≥ 15%
  □ PEG ≤ 1.5
  □ 市值 ≥ 50亿（A股）/ ≥ 100亿港元（港股）

排除条件（触发任一直接剔除）：
  □ ST 股
  □ 上市不足 1 年
  □ 近半年有重大诉讼 / 监管处罚
  □ 近 3 年经营现金流连续为负
  □ 控股股东质押比 > 60%
  □ 商誉/净资产 > 35%
```

### 扫描步骤

**Step 1：全量广度扫描**
- 覆盖 A股（沪深300 + 中证500 重点成分）+ 港股（恒生核心成分）
- 按硬门槛初筛，500–2000只 → 50–150只候选

**Step 2：GARP 初步评分**
- 对候选标的计算 G/Q/估算总分（简化版，不强制完整五因子）
- 📄 评分方法：`docs/garp-core-framework.md` — **五因子评分体系**

**Step 3：候补池增删**
- 新增：估算总分 ≥ 60 且不在现有三个池子里
- 移除：距上次入池超过 3 个月且无任何触发信号
- 容量上限 20 只：超出则保留估算总分最高的 20 只

**Step 4：专项跟踪标的优先复核**
- 对上期标注"待广度扫描验证"的标的优先处理
- 对观察池中标注 A2 预警超过 4 周的标的重新评分确认

**Step 5：完成通知**
- 向 Feishu 群发送月度扫描报告（新入池/移除/候补池总览）

### 候补池→观察池晋升（事件触发）
📄 完整触发条件：`docs/GARP-观察池入选规则-v1.1.md` — **路径B**:事件晋升

快速参考：
```
🔴 强信号（单独触发）：
   - 业绩预告超一致预期 ≥ 20%
   - 重大合同 > 市值 5%
   - 主力净流入 > 30日均值3倍 且同日涨幅 > 3%

🟡 弱信号（任意两个叠加）：
   - 量价双确认异动
   - 赛道 ETF 强/个股滞涨背离
   - 机构净买入单日进前50 或 周内排名升≥10位
   - PEG < 0.3 且近30日跌幅 > 15%（ROE无恶化）
   - 产业链龙头启动后 48h 内未跟随
   - ≥2家机构上调至买入 + 目标价均高于当前 20%+
```

## 五、事件驱动临时扫描

**触发：** 用户指令 / 重大市场行情（指数单日涨跌 > 3%）/ 重大公告
**执行者：** Investor Agent（即时响应）

### 触发场景

| 场景 | 触发条件 | 处理路径 |
|------|----------|---------|
| 重大公告 | 业绩预告/重大合同/管理层变动等 | 判断是否触发强信号，执行对应规则 |
| 市场异动 | 指数单日涨跌 > 3% | 检查 Top15 和观察池的 M 分变化 |
| 外部信号 | 研报/新闻/用户提及 | 先定位标的所在池子，再按对应规则处理 |
| 候补池标的异动 | 满足强信号或两个弱信号叠加 | 晋升观察池 |

### 执行步骤

**Step 1：定位标的所在池子**
- 检查：Top15 / 观察池 / 候补池 / 不在库
- 若不在库：先判断是否满足候补池硬门槛（见下方 **候补池Pipeline** 章节）

**Step 2：按对应规则判断**
- Top15 标的 → 📄 `docs/GARP-观察池入选规则-v1.1.md` — **路径A**:Top15降级
- 观察池标的 → 📄 `docs/GARP-观察池入选规则-v1.1.md` — **双周复查清单**
- 候补池标的 → 📄 `docs/GARP-观察池入选规则-v1.1.md` — **路径B**:事件晋升
- 不在库标的 → 先走候补池硬门槛初筛，通过后做简化 GARP 评分

**Step 3：更新 Bitable 并通知**
- 更新对应池子的 Bitable 记录（触发类型、信号强度、资金流异动等字段）
- 在 Feishu 群发送事件扫描摘要
